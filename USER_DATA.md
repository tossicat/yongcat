# 사용자 용언 데이터 추가

yongcat에 포함되지 않은 용언을 직접 추가할 수 있습니다.

## 사용 방법

### 1. CSV 파일 준비

`data/` 폴더에 CSV 파일을 만듭니다. 파일 이름은 자유이며, 여러 개를 넣어도 됩니다. 단, `yong_list.csv`와 `user_list.csv`는 시스템이 사용하는 이름이므로 사용할 수 없습니다.

```
data/
├── yong_list.csv        ← 기본 데이터 (수정하지 마세요)
├── game_verbs.csv       ← 사용자 파일 1
└── slang.csv            ← 사용자 파일 2
```

### 2. CSV 형식

CSV 형식이며, 모든 컬럼은 문자열입니다. `dict_id`가 `02`처럼 숫자로 보이지만, 앞자리 `0`을 보존해야 하므로 문자열로 취급합니다. 엑셀 등에서 편집할 때 숫자로 자동 변환되지 않도록 주의하세요.

첫 행은 반드시 아래 헤더여야 합니다.

```csv
base_form,dict_id,eogan,pos,conjugation,usage,grade
때리다,,때리,동사,규,,
부수다,,부수,동사,규,,
짜증내다,,짜증내,동사,규,,
```

- **base_form** (필수) — 기본형. "다"로 끝나야 합니다. 예: `때리다`
- **dict_id** (조건부 필수) — 『표준국어대사전』 표제어 일련번호. 같은 기본형의 동음이의어를 구분할 때 사용합니다. 기본 목록(`yong_list.csv`)에 없는 용언이면 비워둬도 됩니다. 기본 목록에 이미 있는 용언의 동음이의어를 추가하려면 반드시 지정해야 합니다. 값은 숫자로 구성된 문자열이며(`"02"`, `"05"` 등), 앞자리 `0`이 보존됩니다. 숫자 타입으로 변환되지 않습니다.
- **eogan** (필수) — 어간. 기본형에서 "다"를 뺀 것과 일치해야 합니다. 예: `때리`
- **pos** (필수) — 품사. `동사` 또는 `형용사`만 허용됩니다.
- **conjugation** (필수) — 활용 유형. 어간이 어미와 결합할 때 어떤 규칙을 따르는지 지정합니다. 대부분의 용언은 `규`(규칙)입니다. 확실하지 않으면 `규`로 지정하세요. 아래 코드표를 참고합니다.
- **usage** (선택) — 의미 설명. 한자어 어원이나 용례를 적습니다.
- **grade** (선택) — 학습 등급. A, B, C 중 하나입니다.

#### 활용 유형 코드

| 코드 | 유형 | 예시 |
|------|------|------|
| 규 | 규칙 | 먹다, 가다 |
| 여 | 여불규칙 | 하다 계열 |
| ㄷ | ㄷ불규칙 | 걷다, 듣다 |
| ㅂ | ㅂ불규칙 | 돕다, 춥다 |
| ㅅ | ㅅ불규칙 | 짓다, 낫다 |
| ㅎ | ㅎ불규칙 | 그렇다, 노랗다 |
| ㄹ | ㄹ불규칙 | 살다, 만들다 |
| 르 | 르불규칙 | 모르다, 빠르다 |
| 우 | 우불규칙 | 푸다 |
| 러 | 러불규칙 | 이르다, 푸르다 |
| 으 | 으불규칙 | 크다, 바쁘다 |

대부분의 용언은 `규`(규칙)입니다. 확실하지 않으면 `규`로 지정하세요.

### 3. 검증 실행

```bash
cargo run --bin import
```

`data/` 폴더의 모든 사용자 CSV 파일을 자동으로 찾아서 검증합니다. (`yong_list.csv`와 `user_list.csv`는 제외됩니다.)

#### 출력 예시

```
=== yongcat 용언 검증 ===

[검증] game_verbs.csv
  ✓ 2행: 때리다 (동사, 규칙)
  ✓ 3행: 부수다 (동사, 규칙)
  ✗ 4행: 오류 — 기본형이 "다"로 끝나지 않음: "때리"

[검증] slang.csv
  ✓ 2행: 꼬시다 (동사, 규칙)
  ✗ 3행: 오류 — "먹다"는 yong_list.csv에 존재합니다. 동음이의어를 추가하려면 dict_id가 필요합니다.

---
결과: 5행 중 3행 통과, 1행 오류, 1행 중복
→ data/user_list.csv 생성 완료 (3행)
```

#### 검증 항목

| 항목 | 설명 |
|------|------|
| 헤더 | 첫 행이 `base_form,dict_id,eogan,pos,conjugation,usage,grade`와 정확히 일치해야 함 |
| 컬럼 수 | 최소 5개 필요 (usage, grade는 선택) |
| 기본형 | "다"로 끝나야 하며, 한글(가~힣)로만 구성되어야 함 |
| 어간 | 기본형에서 "다"를 뺀 것과 일치해야 함 |
| 품사 | `동사` 또는 `형용사`만 허용 |
| 활용 유형 | 위 코드표에 있는 값만 허용 |
| dict_id | 비어 있거나 숫자로만 구성된 문자열 |
| grade | 비어 있거나 A/B/C만 허용 |
| 중복 | `yong_list.csv`에 이미 있는 용언은 dict_id 없이 추가 불가. 동음이의어를 추가하려면 dict_id를 지정해야 함 |

오류가 있는 행은 건너뛰고, 통과한 행만 `data/user_list.csv`에 저장됩니다.

### 4. 빌드

```bash
cargo build
```

`build.rs`가 `yong_list.csv`와 `user_list.csv`를 함께 읽어서 `load_yongeons()`에 포함시킵니다. 이후 `lookup()`, `conjugate()` 등으로 추가한 용언을 바로 사용할 수 있습니다.

```rust
use yongcat::*;

let verb = lookup("때리다");
conjugate(verb, &ASS_SEUMNIDA);  // "때렸습니다"
```

## 파일 구조

```
data/
├── yong_list.csv        # 기본 데이터 (프로젝트 제공, 1,721개)
├── game_verbs.csv       # 사용자 CSV (직접 작성)
├── slang.csv            # 사용자 CSV (직접 작성)
└── user_list.csv        # 검증 통과 데이터 (import가 자동 생성, .gitignore 대상)
```

- `yong_list.csv` — 수정하지 마세요. 프로젝트 업데이트 시 덮어씌워질 수 있습니다.
- `example.csv` — CSV 형식 참고용 예시 파일입니다. 필요 없으면 삭제해도 됩니다.
- `*.csv` (사용자 파일) — 자유롭게 추가/삭제할 수 있습니다.
- `user_list.csv` — `cargo run --bin import`가 자동 생성합니다. 직접 편집하지 마세요.
