# 활용 규칙 구현 가이드

yongcat이 용언 활용형을 생성하는 과정과 각 규칙의 구현 방법을 설명합니다.

## 파이프라인

활용형 생성은 3단계 파이프라인으로 동작합니다.

```
1. 어미 선택 (join)    — 모음조화 판별 → 어미 형태 선택
2. 접합 (join)         — 어간 + 선택된 어미 결합
3. 음운 축약 (merge)   — 모음 축약/탈락 적용
4. 자모 합성 (combine) — 독립 자모를 앞 음절 종성으로 합성
```

`join::select`와 `merge::apply`가 각 단계의 진입점이며, 불규칙이면 `irregular` 모듈에 위임하고, 규칙이면 `regular` 함수로 처리합니다.

```
select ──→ irregular::join ──→ Some이면 반환
           │
           └→ regular        ──→ 모음조화 + 접합

apply  ──→ irregular::merge ──→ Some이면 반환
           │
           └→ regular        ──→ 모음 축약/탈락
```

## 어미 유형별 동작

`Eomi` 열거형은 어미를 세 가지 유형으로 분류합니다. 각 유형에 따라 join 단계의 어미 선택 방식이 달라집니다.

### AhEo — 아/어 계열

모음조화에 따라 형태가 달라지는 어미입니다. `AhEoForm` 튜플에 양성/음성 두 가지 형태를 보관합니다.

```rust
pub type AhEoForm = (&'static str, &'static str);
//                    form.0(양성)   form.1(음성)

pub const AYO: Eomi = Eomi::AhEo(("아요", "어요"));
```

join 단계에서 어간을 보고 `form.0`(양성) 또는 `form.1`(음성)을 선택합니다. 여불규칙은 `form.1`에서 ㅓ→ㅕ 변환으로 여 형태를 생성합니다. merge 단계에서 모음 축약이 적용됩니다.

### Plain — 받침 유무

받침 유무에 따라 형태가 달라지는 어미입니다. 어간에 받침이 있으면 첫 번째, 없으면 두 번째 형태를 선택합니다.

| 어간 | 받침 | 선택 | 예시 |
|------|------|------|------|
| 먹 | 있음 (ㄱ) | "은" | 먹은 |
| 가 | 없음 | "ㄴ" | 간 |

### Fixed — 고정 형태

형태가 고정된 어미입니다. 어간에 그대로 접합합니다.

| 예시 |
|------|
| 먹 + 고 → 먹고 |
| 가 + 고 → 가고 |

## 복합 어미와 선어말 어미

복합 어미(었습니다, 았어요 등)와 선어말 어미를 포함한 형태(겠어요, 으셨 등)는 분리하지 않고, 전체를 하나의 `Eomi` 상수로 정의합니다.

이것이 가능한 이유는 어간과 직접 만나는 첫 음절에서만 변화가 일어나고, 나머지는 항상 고정되기 때문입니다. 파이프라인이 첫 음절만 보고 축약하므로 어미가 얼마나 길든 동작이 같습니다.

```rust
// 복합 어미: 과거 + 합쇼체
pub const ASS_SEUMNIDA: Eomi = Eomi::AhEo(("았습니다", "었습니다"));
//                                          ^^                      ← 어간에 따라 변화
//                                            ^^^^^^                ← 고정

// 선어말 어미 포함: 추측 + 해요체
pub const GESS_EOYO: Eomi = Eomi::Fixed("겠어요");

// 선어말 어미 포함: 높임 과거
pub const EUSYEOSS: Eomi = Eomi::Plain("으셨", "셨");
```

## 음운 법칙

### 모음조화 (join 단계)

양성모음(ㅏ, ㅗ)은 양성모음끼리, 음성모음(ㅓ, ㅜ 등)은 음성모음끼리 어울리는 현상입니다. 아/어 계열 어미(`Eomi::AhEo`)는 어간의 마지막 모음에 맞춰 형태가 바뀝니다.

| 어간 모음 | 선택 | 예시 |
|-----------|------|------|
| ㅏ, ㅗ (양성) | form.0 (아 계열) | 가+아요, 오+아요 |
| 그 외 (음성) | form.1 (어 계열) | 먹+어요, 서+어요 |

구현: `Yongeon::moeum_joha(form)` 메서드가 `is_positive_harmony()`로 판별하여 `form.0` 또는 `form.1`을 반환합니다. `join.rs`의 `regular()`와 불규칙 모듈(ㄷ, ㅅ)이 이 메서드를 공유합니다.

#### ㅡ 어간의 모음조화

어간 마지막 모음이 ㅡ이고 다음절이면, 앞 음절의 모음으로 양성/음성을 판별합니다.

| 용언 | 앞 음절 | 판별 | 결과 |
|------|---------|------|------|
| 바쁘다 | 바 (ㅏ) | 양성 → 아요 | 바빠요 |
| 예쁘다 | 예 (ㅔ) | 음성 → 어요 | 예뻐요 |
| 크다 | (단음절) | 음성 → 어요 | 커요 |

구현: `Yongeon::is_positive_harmony()` 메서드에서 처리합니다.

### 모음 축약 (merge 단계)

어간에 받침이 없을 때, 어간 마지막 모음과 어미 첫 모음이 축약됩니다.

| 규칙 | 어간 모음 | 축약 결과 | 예시 |
|------|-----------|-----------|------|
| 동일모음 탈락 | ㅏ+ㅏ | ㅏ | 가+아요 → 가요 |
| 동일모음 탈락 | ㅓ+ㅓ | ㅓ | 서+어요 → 서요 |
| ㅗ 축약 | ㅗ+ㅏ | ㅘ | 오+아요 → 와요 |
| ㅜ 축약 | ㅜ+ㅓ | ㅝ | 주+어요 → 줘요 |
| ㅣ 축약 | ㅣ+ㅓ | ㅕ | 피+어요 → 펴요 |
| ㅡ 탈락 | ㅡ+어미 | 어미 첫 모음 | 크+어요 → 커요 |

구현: `merge.rs`의 `contract_ah_eo()` 함수에서 어간 마지막 음절의 모음을 변경하고, 어미 첫 음절의 종성(있으면)을 어간으로 이전합니다.

#### 종성 이전

과거 시제 어미(았/었)처럼 어미 첫 음절에 종성이 있으면, 축약 시 그 종성이 어간 마지막 음절로 이전됩니다.

```
가 + 았 → 가(ㅏ) + 았(ㅏ,ㅆ)
        → 모음 축약: ㅏ+ㅏ → ㅏ
        → 종성 이전: ㅆ → 어간으로
        → 갔
```

## 불규칙 활용

### 구조

각 불규칙은 `src/irregular/` 아래 별도 파일로 구현됩니다.

```
src/irregular/
├── mod.rs     — IrregularType별 디스패치
├── yeo.rs     — 여불규칙
├── dieut.rs   — ㄷ불규칙
├── bieut.rs   — ㅂ불규칙
├── siot.rs    — ㅅ불규칙
├── rieul.rs   — ㄹ불규칙
├── u.rs       — 우불규칙
├── hieut.rs   — ㅎ불규칙
├── reo.rs     — 러불규칙
├── reu.rs     — 르불규칙
└── eu.rs      — 으불규칙 (문서만, 규칙 활용으로 처리)
```

각 모듈은 `join`과 `merge` 함수를 제공하며, `Option<String>`을 반환합니다.
- `Some(결과)`: 불규칙 규칙이 적용됨
- `None`: 해당 단계에 개입하지 않음 → 규칙 활용으로 위임

### 새 불규칙 추가 방법

1. `src/irregular/` 아래에 새 파일 생성
2. `join`과 `merge` 함수 구현 (불필요한 단계는 `None` 반환)
3. `mod.rs`에 모듈 선언과 디스패치 추가
4. 유닛 테스트를 모듈 파일에, 통합 테스트를 `tests/postfix_test.rs`에 추가

### 여불규칙

"하다" 계열 용언이 아/어 계열 어미를 만나면 "여" 형태를 사용하고, 하+여 → 해 축약을 적용합니다. CSV 데이터에서 `conjugation=여`로 표시됩니다.

| 단계 | 동작 | 예시 (하다 + 어요) |
|------|------|-------------------|
| join | form.1(음성)의 ㅓ→ㅕ 변환으로 여 형태 생성 | 어요→여요, 하 + 여요 → 하여요 |
| merge | 어간 모음 ㅏ → ㅐ 변경 | 하여요 → 해요 |

- join: `AhEo`이면 `to_yeo(form.1)`로 여 형태 생성, `Plain`/`Fixed`이면 `None`
- merge: `AhEo`이면 ㅏ→ㅐ 축약, `Plain`/`Fixed`이면 `None`

```
하다 + 아요/어요 → join: 어요→여요, 하+여요 → merge: 해요
공부하다 + 았/었 → join: 었→였, 공부하+였 → merge: 공부했
```

### ㄷ불규칙

어간 끝 받침 ㄷ이 모음으로 시작하는 어미 앞에서 ㄹ로 바뀝니다. CSV 데이터에서 `conjugation=ㄷ`으로 표시됩니다.

| 단계 | 동작 | 예시 (걷다 + 어요) |
|------|------|-------------------|
| join | 어간 끝 ㄷ → ㄹ 변환 후 접합 | 걷→걸 + 어요 → 걸어요 |
| merge | 개입 없음 (None) | 걸어요 그대로 |

- join: `AhEo`/`Plain`(모음 시작)이면 ㄷ→ㄹ 변환 후 모음조화에 따라 접합, 자음 시작이면 `None`
- merge: 항상 `None` (어간에 받침 ㄹ이 남아 있어 규칙 merge가 축약 없이 통과)

```
걷다 + 어요 → join: 걸+어요 → merge: 걸어요
듣다 + 었   → join: 들+었   → merge: 들었
```

### ㅂ불규칙

어간 끝 받침 ㅂ이 모음 어미 앞에서 우로 바뀝니다. CSV 데이터에서 `conjugation=ㅂ`으로 표시됩니다.

모음조화가 특수합니다: 돕다/곱다(단음절, ㅗ)만 양성(와), 나머지는 전부 음성(워)입니다.

| 단계 | 동작 | 예시 (돕다 + 아요) |
|------|------|-------------------|
| join | ㅂ 제거 + 우 삽입, 특수 모음조화로 어미 선택 | 돕→도우 + 아요 → 도우아요 |
| merge | 우+어미 첫 모음 축약 (ㅜ+ㅏ→ㅘ, ㅜ+ㅓ→ㅝ) | 도우아요 → 도와요 |

- join: `AhEo`이면 ㅂ→우 변환 후 특수 모음조화에 따라 접합, `Plain`(모음 시작)이면 ㅂ→우 + 무받침 형태 선택, 자음 시작이면 `None`
- merge: `AhEo`이면 우+어미 축약, `Plain`/`Fixed`이면 `None`

```
돕다 + 아요 → join: 도우+아요 → merge: 도와요  (단음절 ㅗ → 양성)
춥다 + 어요 → join: 추우+어요 → merge: 추워요  (음성)
가깝다 + 어요 → join: 가까우+어요 → merge: 가까워요  (다음절 → 음성)
```

### ㅅ불규칙

어간 끝 받침 ㅅ이 모음 어미 앞에서 탈락합니다. CSV 데이터에서 `conjugation=ㅅ`으로 표시됩니다.

규칙 활용과 달리 ㅅ 탈락 후 모음 축약이 적용되지 않습니다 (축약 억제).

| 단계 | 동작 | 예시 (짓다 + 어요) |
|------|------|-------------------|
| join | 어간 끝 ㅅ 제거 후 접합 | 짓→지 + 어요 → 지어요 |
| merge | 축약 억제 (그대로 반환) | 지어요 그대로 (져요가 아님) |

- join: `AhEo`이면 ㅅ 제거 후 모음조화에 따라 접합, `Plain`(모음 시작)이면 ㅅ 제거 + 무받침 형태 선택, 자음 시작이면 `None`
- merge: `AhEo`이면 축약 없이 그대로 반환, `Plain`/`Fixed`이면 `None`

```
짓다 + 어요 → join: 지+어요 → merge: 지어요  (축약 억제)
낫다 + 아요 → join: 나+아요 → merge: 나아요  (축약 억제)
짓다 + 었   → join: 지+었   → merge: 지었    (축약 억제)
```

### ㄹ불규칙

ㄴ/ㅂ/ㅅ 앞에서 어간 끝 ㄹ이 탈락합니다. Plain 어미에서는 항상 무받침 형태를 선택합니다. AhEo 어미에서는 규칙 활용과 동일하므로 개입하지 않습니다. CSV 데이터에서 `conjugation=ㄹ`로 표시됩니다.

| 단계 | 동작 | 예시 (살다 + 는) |
|------|------|-----------------|
| join | ㄴ/ㅂ/ㅅ 앞 ㄹ 탈락, Plain은 무받침 형태 선택 | 살→사 + 는 → 사는 |
| merge | 개입 없음 (None) | — |

```
살다 + 는   → 사는   (Fixed, ㄹ탈락 before ㄴ)
살다 + 면   → 살면   (Plain, ㄹ유지, 무받침형 선택)
살다 + ㅂ니다 → 삽니다  (Plain, ㄹ탈락 before ㅂ, 자모 합성)
살다 + 아요  → 살아요  (AhEo, 개입 없음)
```

### 우불규칙

어간 모음 ㅜ가 아/어 계열 어미 앞에서 어미 첫 모음으로 대체됩니다. ㅡ 탈락과 같은 패턴이지만 ㅜ에 적용됩니다. 푸다 하나만 해당합니다. CSV 데이터에서 `conjugation=우`로 표시됩니다.

| 단계 | 동작 | 예시 (푸다 + 어요) |
|------|------|-------------------|
| join | 개입 없음 (규칙 활용과 동일) | 푸 + 어요 → 푸어요 |
| merge | ㅜ → 어미 첫 모음으로 대체 (ㅝ 축약이 아님) | 푸어요 → 퍼요 |

```
푸다 + 어요 → join: 푸+어요 → merge: 퍼요  (ㅜ→ㅓ)
푸다 + 었   → join: 푸+었   → merge: 펐    (ㅜ→ㅓ, ㅆ 이전)
```

### 러불규칙

아/어 계열 어미의 "어"가 "러"로 바뀝니다. 어간의 ㅡ가 탈락하지 않고 유지됩니다. CSV 데이터에서 `conjugation=러`로 표시됩니다.

| 단계 | 동작 | 예시 (이르다 + 어요) |
|------|------|---------------------|
| join | form.1(음성)의 초성 ㅇ→ㄹ 변환으로 러 형태 생성 | 어요→러요, 이르 + 러요 → 이르러요 |
| merge | 축약 억제 (그대로 반환) | 이르러요 그대로 |

```
이르다 + 어요 → join: 이르+러요 → merge: 이르러요
푸르다 + 어요 → join: 푸르+러요 → merge: 푸르러요
```

### ㅎ불규칙

어간 끝 받침 ㅎ이 모음 어미 앞에서 탈락하고, 아/어 계열 어미와 만나면 어간 끝 모음이 ㅐ(또는 ㅒ)로 축약됩니다. 대상은 모두 형용사(13개)입니다. CSV 데이터에서 `conjugation=ㅎ`으로 표시됩니다.

| 단계 | 동작 | 예시 (그렇다 + 어요) |
|------|------|---------------------|
| join | ㅎ 탈락 후 접합 | 그렇→그러 + 어요 → 그러어요 |
| merge | 어간 끝 모음 → ㅐ 축약, 어미 첫 음절 흡수 | 그러어요 → 그래요 |

- join: `AhEo`이면 ㅎ 제거 후 모음조화에 따라 접합, `Plain`(모음 시작)이면 ㅎ 제거 + 무받침 형태 선택, 자음 시작이면 `None`
- merge: `AhEo`이면 어간 끝 모음을 ㅐ(ㅑ인 경우 ㅒ)로 변경하고 어미 첫 음절의 받침을 흡수, `Plain`/`Fixed`이면 `None`

```
그렇다 + 어요 → join: 그러+어요 → merge: 그래요   (ㅓ→ㅐ)
노랗다 + 아요 → join: 노라+아요 → merge: 노래요   (ㅏ→ㅐ)
하얗다 + 아서 → join: 하야+어서 → merge: 하얘서   (ㅑ→ㅒ)
그렇다 + 었   → join: 그러+었   → merge: 그랬     (ㅐ + ㅆ 흡수)
그렇다 + 으면 → join: 그러+면   → 그러면           (ㅎ 탈락, Plain)
그렇다 + 고   → 그렇고                              (Fixed, 변환 없음)
```

### 르불규칙

어간 끝 르가 모음 어미 앞에서 분리되어, ㄹ이 앞 음절 받침으로 삽입되고 어미 첫 초성이 ㄹ로 바뀝니다. 동사(25개)와 형용사(11개) 모두 해당합니다. CSV 데이터에서 `conjugation=르`로 표시됩니다.

| 단계 | 동작 | 예시 (모르다 + 아요) |
|------|------|---------------------|
| join | 르 제거, 앞 음절에 ㄹ 삽입, 어미 초성 ㄹ 교체 | 모르→몰 + 라요 → 몰라요 |
| merge | 축약 억제 (그대로 반환) | 몰라요 그대로 |

- join: `AhEo`이면 르 분리 + ㄹ 삽입 + 어미 초성 교체, `Plain`/`Fixed`이면 `None`
- merge: `AhEo`이면 축약 없이 그대로 반환, `Plain`/`Fixed`이면 `None`

```
모르다 + 아요 → join: 몰+라요 → merge: 몰라요   (양성, ㅗ)
기르다 + 어요 → join: 길+러요 → merge: 길러요   (음성, ㅣ)
빠르다 + 아요 → join: 빨+라요 → merge: 빨라요   (양성, ㅏ)
떠오르다 + 아요 → join: 떠올+라요 → merge: 떠올라요  (다음절)
모르다 + 았   → join: 몰+랐   → merge: 몰랐     (과거)
모르다 + 면   → 모르면                              (Plain, 변환 없음)
```

### 으불규칙

별도 불규칙 처리가 필요 없습니다. 규칙 활용의 ㅡ 탈락(`contract_ah_eo`)과 앞 음절 모음조화(`is_positive_harmony`)로 처리됩니다. CSV 데이터에서 `conjugation=으`로 표시됩니다.

```
잠그다 + 아요 → 잠가요  (앞 음절 ㅏ 양성)
끄다 + 어요   → 꺼요    (단음절 음성)
슬프다 + 어요 → 슬퍼요  (앞 음절 ㅡ→ㅓ 음성)
```

## 품사별 어미 사용 제한

46개 어미 중 12개는 동사에만 사용할 수 있습니다. 형용사에 적용하면 문법적으로 잘못된 활용형이 생성됩니다.

| 상수 | 어미 | 제한 사유 |

### 형태는 같으나 품사별 기능이 다른 어미

일부 어미는 동사와 형용사 모두 사용 가능하지만, 문법적 기능이 달라집니다.

| 상수 | 어미 | 동사 | 형용사 |
|------|------|------|--------|
| ARA | 아라/어라 | 명령 ("먹어라") | 감탄 ("좋아라!") |
| EUN | 은/ㄴ | 과거 관형사형 ("먹은 밥") | 현재 관형사형 ("예쁜 꽃") |
| EUSEYO | 으세요/세요 | 명령/평서 ("드세요") | 평서/의문 ("예쁘세요") |
| DA | 다 | 기본형 ("먹다") | 기본형 ("예쁘다"), 동사 현재 평서문은 보통 "-ㄴ다/는다" 사용 |

### 현재 구현 상태

`conjugate()`는 품사 제한을 검사하지 않습니다. 안전한 활용형 생성이 필요하면 `conjugate_checked()`를 사용합니다.

```rust
// conjugate(): 제한 없음 (잘못된 활용형이 생성될 수 있음)
conjugate(lookup("예쁘다"), &NEUN)  // → "예쁘는" (틀림)

// conjugate_checked(): 품사 제한 검사
conjugate_checked(lookup("예쁘다"), &NEUN)  // → Err("형용사 \"예쁘다\"에 동사 전용 어미를 적용할 수 없습니다")
conjugate_checked(lookup("먹다"), &NEUN)    // → Ok("먹는")
```
