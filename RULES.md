# 활용 규칙 구현 가이드

yongcat이 용언 활용형을 생성하는 과정과 각 규칙의 구현 방법을 설명합니다.

## 파이프라인

활용형 생성은 3단계 파이프라인으로 동작합니다.

```
1. 어미 선택 (join)    — 모음조화 판별 → 어미 형태 선택
2. 접합 (join)         — 어간 + 선택된 어미 결합
3. 음운 축약 (merge)   — 모음 축약/탈락 적용
```

`join::select`와 `merge::apply`가 각 단계의 진입점이며, 불규칙이면 `irregular` 모듈에 위임하고, 규칙이면 `regular` 함수로 처리합니다.

```
select ──→ irregular::join ──→ Some이면 반환
           │
           └→ regular        ──→ 모음조화 + 접합

apply  ──→ irregular::merge ──→ Some이면 반환
           │
           └→ regular        ──→ 모음 축약/탈락
```

## 어미 유형별 동작

`Eomi` 열거형은 어미를 세 가지 유형으로 분류합니다. 각 유형에 따라 join 단계의 어미 선택 방식이 달라집니다.

### AhEo — 아/어 계열

모음조화에 따라 형태가 달라지는 어미입니다. `AhEoForm` 튜플에 양성/음성 두 가지 형태를 보관합니다.

```rust
pub type AhEoForm = (&'static str, &'static str);
//                    form.0(양성)   form.1(음성)

pub const AYO: Eomi = Eomi::AhEo(("아요", "어요"));
```

join 단계에서 어간을 보고 `form.0`(양성) 또는 `form.1`(음성)을 선택합니다. 여불규칙은 `form.1`에서 ㅓ→ㅕ 변환으로 여 형태를 생성합니다. merge 단계에서 모음 축약이 적용됩니다.

### Plain — 받침 유무

받침 유무에 따라 형태가 달라지는 어미입니다. 어간에 받침이 있으면 첫 번째, 없으면 두 번째 형태를 선택합니다.

| 어간 | 받침 | 선택 | 예시 |
|------|------|------|------|
| 먹 | 있음 (ㄱ) | "은" | 먹은 |
| 가 | 없음 | "ㄴ" | 가ㄴ |

### Fixed — 고정 형태

형태가 고정된 어미입니다. 어간에 그대로 접합합니다.

| 예시 |
|------|
| 먹 + 고 → 먹고 |
| 가 + 고 → 가고 |

## 음운 법칙

### 모음조화 (join 단계)

양성모음(ㅏ, ㅗ)은 양성모음끼리, 음성모음(ㅓ, ㅜ 등)은 음성모음끼리 어울리는 현상입니다. 아/어 계열 어미(`Eomi::AhEo`)는 어간의 마지막 모음에 맞춰 형태가 바뀝니다.

| 어간 모음 | 선택 | 예시 |
|-----------|------|------|
| ㅏ, ㅗ (양성) | form.0 (아 계열) | 가+아요, 오+아요 |
| 그 외 (음성) | form.1 (어 계열) | 먹+어요, 서+어요 |

구현: `join.rs`의 `regular()` 함수에서 `yongeon.is_positive_harmony()`로 판별합니다.

#### ㅡ 어간의 모음조화

어간 마지막 모음이 ㅡ이고 다음절이면, 앞 음절의 모음으로 양성/음성을 판별합니다.

| 용언 | 앞 음절 | 판별 | 결과 |
|------|---------|------|------|
| 바쁘다 | 바 (ㅏ) | 양성 → 아요 | 바빠요 |
| 예쁘다 | 예 (ㅔ) | 음성 → 어요 | 예뻐요 |
| 크다 | (단음절) | 음성 → 어요 | 커요 |

구현: `Yongeon::is_positive_harmony()` 메서드에서 처리합니다.

### 모음 축약 (merge 단계)

어간에 받침이 없을 때, 어간 마지막 모음과 어미 첫 모음이 축약됩니다.

| 규칙 | 어간 모음 | 축약 결과 | 예시 |
|------|-----------|-----------|------|
| 동일모음 탈락 | ㅏ+ㅏ | ㅏ | 가+아요 → 가요 |
| 동일모음 탈락 | ㅓ+ㅓ | ㅓ | 서+어요 → 서요 |
| ㅗ 축약 | ㅗ+ㅏ | ㅘ | 오+아요 → 와요 |
| ㅜ 축약 | ㅜ+ㅓ | ㅝ | 주+어요 → 줘요 |
| ㅣ 축약 | ㅣ+ㅓ | ㅕ | 피+어요 → 펴요 |
| ㅡ 탈락 | ㅡ+어미 | 어미 첫 모음 | 크+어요 → 커요 |

구현: `merge.rs`의 `contract_ah_eo()` 함수에서 어간 마지막 음절의 모음을 변경하고, 어미 첫 음절의 종성(있으면)을 어간으로 이전합니다.

#### 종성 이전

과거 시제 어미(았/었)처럼 어미 첫 음절에 종성이 있으면, 축약 시 그 종성이 어간 마지막 음절로 이전됩니다.

```
가 + 았 → 가(ㅏ) + 았(ㅏ,ㅆ)
        → 모음 축약: ㅏ+ㅏ → ㅏ
        → 종성 이전: ㅆ → 어간으로
        → 갔
```

## 불규칙 활용

### 구조

각 불규칙은 `src/irregular/` 아래 별도 파일로 구현됩니다.

```
src/irregular/
├── mod.rs     — IrregularType별 디스패치
├── yeo.rs     — 여불규칙
└── dieut.rs   — ㄷ불규칙
```

각 모듈은 `join`과 `merge` 함수를 제공하며, `Option<String>`을 반환합니다.
- `Some(결과)`: 불규칙 규칙이 적용됨
- `None`: 해당 단계에 개입하지 않음 → 규칙 활용으로 위임

### 새 불규칙 추가 방법

1. `src/irregular/` 아래에 새 파일 생성 (예: `bieut.rs`)
2. `join`과 `merge` 함수 구현 (불필요한 단계는 `None` 반환)
3. `mod.rs`에 모듈 선언과 디스패치 추가
4. 유닛 테스트를 모듈 파일에, 통합 테스트를 `tests/postfix_test.rs`에 추가

### 여불규칙

"하다" 계열 용언이 아/어 계열 어미를 만나면 "여" 형태를 사용하고, 하+여 → 해 축약을 적용합니다. CSV 데이터에서 `conjugation=여`로 표시됩니다.

| 단계 | 동작 | 예시 (하다 + 어요) |
|------|------|-------------------|
| join | form.1(음성)의 ㅓ→ㅕ 변환으로 여 형태 생성 | 어요→여요, 하 + 여요 → 하여요 |
| merge | 어간 모음 ㅏ → ㅐ 변경 | 하여요 → 해요 |

- join: `AhEo`이면 `to_yeo(form.1)`로 여 형태 생성, `Plain`/`Fixed`이면 `None`
- merge: `AhEo`이면 ㅏ→ㅐ 축약, `Plain`/`Fixed`이면 `None`

```
하다 + 아요/어요 → join: 어요→여요, 하+여요 → merge: 해요
공부하다 + 았/었 → join: 었→였, 공부하+였 → merge: 공부했
```

### ㄷ불규칙

어간 끝 받침 ㄷ이 모음으로 시작하는 어미 앞에서 ㄹ로 바뀝니다. CSV 데이터에서 `conjugation=ㄷ`으로 표시됩니다.

| 단계 | 동작 | 예시 (걷다 + 어요) |
|------|------|-------------------|
| join | 어간 끝 ㄷ → ㄹ 변환 후 접합 | 걷→걸 + 어요 → 걸어요 |
| merge | 개입 없음 (None) | 걸어요 그대로 |

- join: `AhEo`이면 ㄷ→ㄹ 변환 후 모음조화에 따라 접합, `Plain`/`Fixed`이면 `None`
- merge: 항상 `None` (어간에 받침 ㄹ이 남아 있어 규칙 merge가 축약 없이 통과)

```
걷다 + 어요 → join: 걸+어요 → merge: 걸어요
듣다 + 었   → join: 들+었   → merge: 들었
```

### ㅂ불규칙

어간 끝 받침 ㅂ이 모음 어미 앞에서 우로 바뀝니다. CSV 데이터에서 `conjugation=ㅂ`으로 표시됩니다.

모음조화가 특수합니다: 돕다/곱다(단음절, ㅗ)만 양성(와), 나머지는 전부 음성(워)입니다.

| 단계 | 동작 | 예시 (돕다 + 아요) |
|------|------|-------------------|
| join | ㅂ 제거 + 우 삽입, 특수 모음조화로 어미 선택 | 돕→도우 + 아요 → 도우아요 |
| merge | 우+어미 첫 모음 축약 (ㅜ+ㅏ→ㅘ, ㅜ+ㅓ→ㅝ) | 도우아요 → 도와요 |

- join: `AhEo`이면 ㅂ→우 변환 후 특수 모음조화에 따라 접합, `Plain`/`Fixed`이면 `None`
- merge: `AhEo`이면 우+어미 축약, `Plain`/`Fixed`이면 `None`

```
돕다 + 아요 → join: 도우+아요 → merge: 도와요  (단음절 ㅗ → 양성)
춥다 + 어요 → join: 추우+어요 → merge: 추워요  (음성)
가깝다 + 어요 → join: 가까우+어요 → merge: 가까워요  (다음절 → 음성)
```

## 미구현 불규칙

| 유형 | join 예상 동작 | merge 예상 동작 |
|------|---------------|----------------|
| ㅅ불규칙 | ㅅ 탈락 | 축약 억제 (지+어요 → 지어요) |
| ㅎ불규칙 | ㅎ 탈락 | 특수 축약 (ㅐ 또는 ㅔ, 구현 시 검증 필요) |
| ㄹ불규칙 | Plain 어미에서 ㄹ 탈락 | — |
| 르불규칙 | ㄹ 삽입 + 어간 변환 | 특수 |
| 우불규칙 | 어간 변환 | 특수 |
| 러불규칙 | 어미 변환 | — |
| 으불규칙 | 어간 변환 | — |
